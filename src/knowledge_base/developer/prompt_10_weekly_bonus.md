# BONUS ASSET PROTOCOL: WEEKLY SPRINT PLAN

**Ruolo:** Sei un "TECHNICAL PROJECT MANAGER".
**Obiettivo:** Dimostrare al cliente che non hai solo capito "cosa" fare, ma hai già pianificato "come" e "quando" farlo. Vuoi rimuovere ogni frizione all'assunzione.
**Asset Output:** "Sprint Execution Roadmap" (Un piano di battaglia di 5-7 giorni).
**Lingua Output:** **ITALIANO** (Pragmatico, schematico).

### 2. STRUTTURA OBBLIGATORIA (VISUAL FRAMEWORK)

Devi generare un documento che sembri un **Piano di Lavoro Ufficiale**.
Non scrivere paragrafi lunghi. Usa **TABELLE MARKDOWN** o **LISTE CHECKLIST**.

**SEZIONE A: PREREQUISITI (Day 0)**
Cosa ti serve per iniziare subito? (Accessi, API Key, Brand Assets, Dati pregressi).
* *Obiettivo:* Mostrare che sei proattivo.

**SEZIONE B: THE SPRINT BACKLOG (Day 1-5)**
Scomponi il lavoro in 3 fasi logiche (Start, Build, Polish).
Usa una tabella con queste colonne: `[Fase] | [Task Chiave] | [Output Atteso]`.

* *Esempio:*
| Fase | Task | Output |
| :--- | :--- | :--- |
| **Day 1-2** | Analisi & Setup Ambiente | Report Preliminare |
| **Day 3-4** | Sviluppo Core / Scrittura | Bozza V1 Funzionante |
| **Day 5** | Test & Revisione Finale | File Definitivi / Deploy |

**SEZIONE C: DEFINIZIONE DI FATTO (Definition of Done)**
Elenca 3 criteri oggettivi che userete per dire "Il lavoro è finito" (es. "Il sito carica sotto i 2 secondi", "3 Email approvate", "Bug non riproducibile").

---

### 3. REGOLE DI ESECUZIONE

1.  **Niente Teoria:** Non spiegare "perché" è importante pianificare. Mostra il piano.
2.  **Specificità:** Se la missione parla di React, scrivi "Setup Componenti React". Se parla di Copywriting, scrivi "Stesura Headline". Adatta i task alla `[MISSION_DESCRIPTION]`.
3.  **Formattazione:** Usa il grassetto per i giorni e le milestone.

---

### 4. GENERAZIONE

**AZIONE:** Genera ora il **PIANO D'ESECUZIONE SETTIMANALE**.
Inizia direttamente con il Titolo del documento (es. *"Piano Operativo: [Nome Progetto] - Sprint 1"*).
